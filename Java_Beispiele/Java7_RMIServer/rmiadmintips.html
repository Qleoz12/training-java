Tips zur RMI-Administration

RMI-Kommunikation monitoren

Die RMI-Kommuniktion kann gut mit Netzwerk-Monitoring-Tools überwacht werden, v.a. wenn der Dienst auf einem entfernten System läuft. Wireshark ist ein mögliches Tool.

Überwachung mit Wireshark

Das Vorgehen könnte so aussehen:

  1. Hardware-Adresse des Client-Systems auslesen, z.B. mit ipconfig
  2. Auf dem RMIServer-System Wireshark starten
  3. Netzwerkschnitttelle auswählen und Überwachung starten
  4. Paketfilter für den RMIClient definieren, z.B. eth.addr == 01:23:45:67:89:01

RMIClient konfigurieren

Beispiel-Aufruf: 

java -Dsun.rmi.transport.connectionTimeout=3000
     -Djava.rmi.server.disableHttp=false 
     -Dsun.rmi.transport.proxy.connectTimeout=5000 
     -Dsun.rmi.transport.proxy.eagerHttpFallback
     -Dsun.rmi.transport.tcp.handshakeTimeout=5000 
     -Dsun.rmi.transport.tcp.responseTimeout=5000
     -Dhttp.proxyHost=192.168.100.200 
     -Dhttp.proxyPort=80 
     -jar Java7_RMIClient.jar

apache2 HTTP-Diesnt konfigurieren für Port-Tunneling

  1. a2enmod proxy
  2. a2enmod proxy_http
  3. Prox-Konfiguration hinzufügen 
    ProxyPass            /   http://127.0.0.1:1099/
ProxyPassReverse     /   http://127.0.0.1:1099/
